2. WORKFLOW API (Same host as provision — va_golang_infra_provisionner, Port 8743)
================================================================================
Workflow URL (from main.go workflow.RegisterRoutes): http://localhost:8743/api/v2/workflow

All provisioning actions (EC2, EKS, AKS, GKE, Docker, RDS, static site, FastAPI, etc.) are executed by sending a workflow to /execute. The frontend maps node types to resource_type and cloud_provider.

--- 2.1 Execute workflow (main provisioning endpoint)
  Method: POST
  Route: /execute
  Payload: {
    "nodes": Array<{ id, type, position, data: { label?, resource_type?, cloud_provider?, config? } }>,  // mapped from frontend nodes
    "edges": Array<{ id, source, target, type, data? }>,
    "metadata": { name?, description?, error_handling?, retry_attempts?, version? },
    "terraform_code"?: string,      // optional frontend-generated HCL
    "use_frontend_terraform"?: boolean

  Response: ExecutionResult { success, execution_id?, logs, status, error?, node_results?, duration? }

  Method: POST
  Route: /save
  Payload: { name, nodes, edges, metadata, terraform_code }
  Response: { success, session_id?, terraform_dir?, error? }

  Route: /validate
  Payload: { nodes, edges, metadata }

5. INTENT-TO-RESOURCE MAPPING (Frontend workflow_service.ts — for workflow/execute)
================================================================================
When using POST /api/v2/workflow/execute (alternative to direct provision), backend receives workflow definition. User phrases map to intents:

  provision_vm (EC2, Azure VM, GCP Compute):
    EC2 Instance / Virtual Machine -> aws_instance, aws, provision_vm { instance_type: t3.micro, os: ubuntu }
    Azure VM -> azurerm_virtual_machine, azure, provision_vm { vm_size: Standard_B2s }
    GCP Compute -> google_compute_instance, gcp, provision_vm { machine_type: e2-micro }

  provision_kubernetes:
    Kubernetes Cluster / EKS Cluster -> aws_eks_cluster, aws, provision_kubernetes { kubernetes_version: 1.28, node_count: 2 }
    AKS Cluster -> azurerm_kubernetes_cluster, azure, provision_kubernetes
    GKE Cluster -> google_container_cluster, gcp, provision_kubernetes

  provision_database:
    PostgreSQL / MySQL / RDS Instance -> aws_db_instance, aws, provision_database { engine: postgres|mysql, engine_version }
    Aurora -> aws_rds_cluster, aws, provision_database { engine: aurora-postgresql }

  provision_docker:
    Docker Container / Docker -> docker_container, multi, provision_docker { restart_policy: unless-stopped }
    ECS Cluster -> aws_ecs_cluster, aws, provision_ecs { launch_type: FARGATE }

  provision_static_website / provision_fastapi:
    Handled via app node (hostname, app_port, server_name).

  Networking (observability/troubleshooting): VPC, Subnet, Security Group, Route53, Load Balancer, VPN, etc. — resource types and intents (provision_network, provision_dns, provision_vpn, ...) are in workflow node library; for "why can't I access my website" use observability agent and /agent/observability/root-cause or /agent/chat with agent: observability.

================================================================================

Training data: cloud_observability.csv.
  - Cost, coding, budget -> agent: cost or code; use AI service (8741) /agent/chat with respective agent.
  - Summary: one workflow host (8743) for workflow + provision + observability; intents go to the correct path on that host.

--- 12.2 VPN Provisioning (OpenVPN / WireGuard)
  Install endpoints (multipart/form-data):
  POST /api/v1/infrastructure/services/vpn/openvpn/install
  POST /api/v1/infrastructure/services/vpn/wireguard/install
  Required form fields: hostname, ssh_username, username, key_pair_name
  Optional form fields: port, protocol, dns, client_name, session_id

  VPN protocol-to-port mapping:
    openvpn -> 1194 (UDP)
    wireguard -> 51820 (UDP)

  VPN Status/Management:
  POST /api/v1/infrastructure/services/vpn/status
  POST /api/v1/infrastructure/services/vpn/action (actions: start, stop, restart)
  POST /api/v1/infrastructure/services/vpn/download-config
  POST /api/v1/infrastructure/services/vpn/client/add
